1 Lógica de Color

Uso del Navy (#0A192F)

  • Fondos Totales: Exclusivo para Portadas y Separadores de Sección.
  • Texto Principal: Títulos (H1, H2, H3) y negritas sobre fondo blanco.
  • Elementos de UI: Bordes de tablas, líneas gruesas.
  • Prohibido: Usar como fondo en páginas de lectura densa (cansa la vista al imprimir).

Uso del Cyan (#64FFDA)

  • Acentos (Máx 10%): Viñetas (bullets), líneas decorativas delgadas.
  • Enlaces: Hipervínculos en documentos digitales.
  • Énfasis: Iconos pequeños o números destacados.
  • Prohibido: Usar para texto de lectura (poco contraste sobre blanco) o fondos completos.

3 Cuerpo y Espaciado

Normal (Spacing)

Lectura General

Este es el estilo estándar para párrafos de texto. Utilizamos un interlineado de 1.5 o 1.6 (leading-relaxed) para facilitar la lectura en documentos largos.

Entre párrafos debe haber un espacio visible (margin-bottom) equivalente a una línea en blanco. Esto crea "aire" y reduce la fatiga visual.

Font: Roboto Regular 10/11pt | Line-height: 1.5 | Space After: 10pt

Compacto (No Spacing)

Tablas / Listas

Este estilo se utiliza dentro de tablas, notas al pie o listas técnicas donde el espacio es limitado.

El interlineado se reduce a 1.1 o 1.2 (leading-tight) y no hay margen extra entre párrafos, salvo que se indique explícitamente.

Ideal para datos densos o especificaciones técnicas.

Font: Roboto Regular 9/10pt | Line-height: 1.1 | Space After: 0pt

Kryoss no vende "reparación de computadoras" (commodity). Kryoss vende Continuidad Operativa.

"El cliente paga para que su tecnología sea invisible. Si funciona, facturan. Si facturan, nos pagan."

7.1.

Definición del Servicio

Este documento regula la relación entre Kryoss (El Proveedor) y el Cliente. Al aceptar el servicio, el Cliente acepta estas condiciones.

Naturaleza del Servicio

Kryoss actúa como proveedor de servicios de gestión de TI. No somos empleados del Cliente ni tenemos relación de dependencia.

7.2.

Exclusión de Responsabilidad (Software Ilegal)

IMPORTANTE
🏴‍☠️ Responsabilidad del Cliente

El Cliente garantiza poseer las licencias correspondientes para todo el software instalado.

🚫 Renuncia de Garantía

Kryoss NO brindará soporte ni garantía en equipos con Windows/Office activados ilegalmente (Cracks, KMS).

Acción Preventiva

En equipos irregulares, deshabilitaremos actualizaciones automáticas para prevenir bloqueos, bajo responsabilidad del Cliente.

7.3.

Limitación de Responsabilidad (Ciberseguridad)

Aunque utilizamos herramientas de clase mundial, ninguna solución es infalible.

⚡ Fuerza Mayor

Kryoss no responde por daños, lucro cesante o pérdida de datos causados por Ransomware, Malware Día Cero o acciones deliberadas de empleados (Phishing) que eludan protecciones.

💾 Recuperación

En caso de desastre, nuestra responsabilidad se limita exclusivamente a restaurar la operatividad utilizando los backups disponibles.

7.4.

Confidencialidad y Protección de Datos (NDA)

Compromiso

Confidencialidad estricta sobre toda la información (Bases de datos, finanzas).

Propiedad

Los datos son propiedad exclusiva del Cliente. Kryoss no tiene derechos sobre ellos.

Devolución

Al finalizar, eliminamos todos los accesos remotos y credenciales.

KRYOSS
